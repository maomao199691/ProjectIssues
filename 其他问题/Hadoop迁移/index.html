    具体迁移

    迁移namenode

    1. 先关闭依赖hdfs的组件，hive,flume

    2. 关闭hdfs

    3. 修改配置。修改hdfs-site.xml的dfs.namenode.http-address，dfs.namenode.servicerpc-address，core-site.xml的fs.defaultFS到新的节点。

    ```xml dfs.namenode.http-address hadoop10:50070 dfs.namenode.servicerpc-address hadoop10:8020 必须和dfs.namenode.http-address相同机器.

    fs.defaultFS hdfs://hadoop10:9000

    ```

    1. 迁移数据

    修改dfs.namenode.name.dir配置下的文件到新的节点，且文件位置相同。

    1. 同步配置

    迁移SecondaryNameNode

    迁移resourceManager

    1. 关闭服务

    2. 修改配置，修改到新节点。

    ```xml yarn.resourcemanager.hostname hadoop7 

           <property>
               <name>yarn.resourcemanager.address</name>
               <value>hadoop7:8032</value>
       </property>
       <property>
               <name>yarn.resourcemanager.webapp.address</name>
               <value>hadoop7:8088</value>
       </property>

    ```

    1. 启动服务，resourceManager只能在主节点启动服务。

    增加节点

    1. 复制软件包到相同的目录。复制环境配置。

    2. 修改core-site.xml的hadoop.tmp.dir：/data/app/hadoop/hadoopDatas/tempDatas

    3. ```xml hdfs-site.xml dfs.namenode.name.dir /data/app/hadoop/hadoopDatas/namenodeDatas dfs.datanode.data.dir /data/app/hadoop/hadoopDatas/datanodeDatas dfs.namenode.edits.dir /data/app/hadoop/hadoopDatas/dfs/nn/edits dfs.namenode.checkpoint.dir /data/app/hadoop/hadoopDatas/dfs/snn/name dfs.namenode.checkpoint.edits.dir /data/app/hadoop/hadoopDatas/dfs/nn/snn/edits

      ```

      yarn-site.xml

      ```xml yarn.nodemanager.local-dirs /data/app/hadoop/logs/yarn 

            <property>
              <name>yarn.nodemanager.log-dirs</name>
              <value>/data/app/hadoop/logs/yarn/log</value>
      </property>

      ```

    1. 重新启动
    exec "${HADOOP_HDFS_HOME}/bin/hdfs" --config "${HADOOP_CONF_DIR}" --daemon start balancer "$@"

    剔除节点

    加入黑名单只会让其有退役状态，然后复制数据，重新达到三副本状态。退役完成后，从剔除白名单里去掉节点即可。

    1. 创建黑名单,将需要剔除的节点放入配置，配置完成后同步到各个节点。

    ```shell vim blacklist

    echo hadoop8 >>blacklist ```

    在hdfs-site.xml增加配置

    xml <property> <name>dfs.hosts.exclude</name> <value>绝对路径blacklist</value> </property>

    1. 同步完后刷新一下节点，

    shell hdfs dfsadmin -refreshNodes yarn rmadmin -refreshNodes

    刷新完后等待退役，Decommissioning表示节点退役中，Decommissioned退役完成。

    1. 增加白名单，加入所有的节点。然后同步一下配置

    shell hdfs dfsadmin -refreshNodes

    1. 删掉名单里需要剔除的节点。然后刷新一下

    shell hdfs dfsadmin -refreshNodes

    1. 退役完成后关闭节点。

    shell hdfs --daemon stop datanode yarn --daemon stop nodemanager

    1. 关闭nodemanager

    shell hadoop-daemon.sh stop nodemanager

    再次加节点

    1. 重复上面加节点操作。

    2. 启动nodemanager,检查resourcemanager的资源节点是否增加

    shell yarn-daemon.sh start nodemanager

    1. 启动datanode

    shell hadoop-daemon.sh start datanode

    1. 修改配置

    节点加入workers,节点加入白名单，同步配置

    1. 刷新集群节点

    shell hdfs dfsadmin -refreshNodes
